# オンボーディング（初回説明スライド）機能 提案書

アプリ初回起動時に表示される、アプリのコンセプトと使い方を説明するスライド機能（オンボーディング）の提案です。

## 1. コンテンツ案 (スライド構成)

「脳内ピアノ（Brain Piano）」という独自のコンセプトを直感的に伝え、ユーザーがスムーズにトレーニングを開始できるように構成します。

| スライド | タイトル | 内容・メッセージ | ビジュアルイメージ |
| :--- | :--- | :--- | :--- |
| **1** | **ようこそ 8va へ** | 「脳内ピアノ」で鍛える、新しい相対音感トレーニング。<br>あなたの歌声がコントローラーです。 | アプリのロゴと、脳内に鍵盤が浮かんでいるようなイラスト |
| **2** | **歌って答える** | 音名（ドレミ）を答える必要はありません。<br>聞こえた音の「相対的な位置」をイメージして、声に出して歌うだけ。 | マイクに向かって歌うユーザーと、反応するアプリ画面 |
| **3** | **脳内ピアノ理論** | 絶対音感は不要です。<br>基準音（ド）から、次の音がどの鍵盤にあるか。<br>その「距離感」を脳内で可視化します。 | 鍵盤のイラストと、音程の距離を示す矢印 |
| **4** | **さあ、始めよう** | マイクの許可をオンにして、<br>あなたの音感の限界に挑戦しましょう。 | 「トレーニングを開始する」ボタン（CTA） |

## 2. 技術的実装案

外部ライブラリ（Swiper.js等）は使用せず、**Vanilla JS + CSS** で軽量に実装することを推奨します。

### A. データ管理
*   `localStorage` を使用して、初回表示済みかどうかを管理します。
    *   Key: `hasSeenOnboarding`
    *   Value: `'true'`

### B. UIコンポーネント構成
`index.html` にモーダルとして追加し、専用のCSS/JSで制御します。

1.  **HTML:** `div.onboarding-modal` (オーバーレイ)
2.  **CSS:** `styles/onboarding.css` (新規作成)
    *   CSS Scroll Snap を使用したスムーズなスワイプ動作
    *   ページネーション（下部のドット）
    *   アニメーション（フェードイン・スライド）
3.  **JS:** `js/components/onboarding-controller.js` (新規作成)
    *   初回起動チェック
    *   スライド切り替えロジック
    *   「開始」ボタン押下時の処理（マイク許可フローへの誘導など）

### C. デザイン方針
*   **没入感:** 全画面オーバーレイで、背景を少し暗くしてスライドに集中させる。
*   **プレミアム感:** アプリ全体のトーンに合わせて、ダークモード基調の洗練されたデザイン。
*   **インタラクション:** スムーズなスライド遷移と、最後の「開始」ボタンの強調。

## 3. 実装ステップ

1.  **画像素材の準備:** 各スライド用のイメージ画像（またはCSSで表現する図形）を作成。
    *   *今回はCSSとLucideアイコン、既存のアセットで表現する形を提案します。*
2.  **コーディング:** HTML/CSS/JSの実装。
3.  **組み込み:** `index.html` への追加とロード処理。
4.  **動作確認:** 初回起動時のみ表示されるか、閉じた後は表示されないか確認。
